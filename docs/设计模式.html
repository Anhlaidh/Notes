设计模式

UML

  • UML2.2

    • 结构式图形:强调的是系统式的建模

      • 静态图(类图,对象图,包图)

      • 实现图(组件图,部署图)

      • 剖面图

      • 复合结构图

    • 行为式图形:强调系统模型中触发的事件

      • 活动图

      • 状态图

      • 用例图

    • 交互式图形:属于行为式图形子集合,强调系统模型中资料流程

      • 通信图

      • 交互概述图(UML2.0)

      • 时序图(UML2.0)

      • 时间图(UML2.0)

  • 类图

    • Class Diagram:用于表示类,接口,示例等之间相互的静态关系

    • 虽然叫做类图,但类图中不只有类

    1.箭头子类指向父类
    2.实线-继承 虚线-实现
    3. 空心三角箭头：继承或者实现
    4. 实线-继承 5. 虚线-实现

设计模式

单例

  • 饿汉式,类加载的同时初始化实例对象,static final,new,使用getInstance方法来调用

  • 懒汉式,把实例对象通过getInstance方法来实例,if(null) else instance
